# SAT Prep Master

AI-powered SAT practice app that generates unique, adaptive questions using Claude and tracks your performance over time.

## Features

- **Full Adaptive Test** — mirrors the real digital SAT: two R&W modules (27 questions, 32 min each) and two Math modules (22 questions, 35 min each). Module 2 difficulty adapts based on your Module 1 score (≥70% → harder pool).
- **Custom Practice** — choose section, topic, difficulty, and question count for targeted drilling.
- **AI Question Generation** — every question is freshly generated by Claude. Math questions maintain the real SAT's ~75% multiple-choice / ~25% grid-in split.
- **Harder-than-real calibration** — questions are intentionally one tier harder than the real SAT so test day feels easier.
- **Realistic scoring** — raw scores convert to 200–800 scaled scores using a lookup table modeled on College Board's published conversion curves, with a difficulty bonus applied so estimates reflect real-exam performance.
- **Dashboard** — score history chart, accuracy by topic, weakest topic identification.
- **Full test review** — every past test is saved and revisitable, question by question.

## Setup

### Prerequisites
- Node.js 18+
- An Anthropic API key ([console.anthropic.com](https://console.anthropic.com))

### 1. Backend

```bash
cd backend
npm install
cp .env.example .env
# Edit .env and set ANTHROPIC_API_KEY and JWT_SECRET
npm run dev
```

The API starts on `http://localhost:3001`. SQLite database is auto-created at `backend/data/sat_prep.db`.

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
```

Open `http://localhost:5173`.

| Variable | Description |
|---|---|

| `JWT_SECRET` | Any long random string for signing JWTs |

## How It Works

### Adaptive Difficulty
After Module 1, the backend scores your answers. If accuracy ≥ 70%, Module 2 questions are generated from the "harder" pool; otherwise the "easier" pool. This mirrors College Board's adaptive routing.

### Question Generation
Questions are generated in batches of ~9 via the Claude API. The system prompt instructs Claude to calibrate all questions one tier harder than the stated difficulty label:
- "easy" → real SAT medium difficulty
- "medium" → real SAT hard
- "hard" → real SAT's hardest (top 5% territory)

Responses are validated for schema correctness and retried up to 2 times on malformed output.

### Scoring
Raw-to-scaled conversion uses lookup tables modeled on College Board's released SAT score conversion charts (visible in Bluebook practice test answer keys). The curve is non-linear: near-perfect scores lose very little per missed question, the middle range costs more, and the bottom compresses toward 200.

A difficulty bonus (+5 questions for R&W, +4 for Math) is applied to the adjusted raw score before lookup to account for the intentionally harder question calibration, giving you a realistic estimate of your real-exam performance.

**Important:** College Board's exact internal curve varies slightly test to test and is not fully public. This is an approximation.

### Weakness Tracking
After enough data accumulates, the dashboard identifies topics with the lowest accuracy (minimum 3 attempts) and surfaces them as focus areas. The "Practice Weak Topics" button pre-selects Custom Practice mode so you can drill those skills directly.

### Grid-In Answer Validation
Math grid-in answers accept integers, decimals, and fractions. The validator normalizes all forms to a float and checks equivalence within ±0.001 (e.g., `1/2`, `0.5`, and `.5` are all accepted). Negative answers are not valid per real SAT rules.
